# Escalation Rules
_Owner: Gary V_
_Last updated: 2026-04-10_

---

## Purpose

Define when Gary V stops and asks Humberto. The system should run without bothering Humberto on routine work — but some decisions need him. These rules define the line.

---

## Always escalate to Humberto

| Trigger | Reason |
|---------|--------|
| Content mentions a specific third party (musician, student, organization, business) | Humberto must approve any public reference to a real person or org |
| Content involves a new or updated offer (price, new service, new program) | Humberto owns offer decisions |
| Content is the first post of a new campaign | Campaign launches should have Humberto's eyes on them |
| Content involves personal or sensitive subject matter (illness, conflict, political topic) | Not Gary V's call |
| A post makes a factual claim about an event, organization, or person | Fact-checks require Humberto's knowledge |
| Scheduling or budget decisions that affect partnerships or events | Operational decisions with real-world consequences |
| Any content that could embarrass Humberto professionally if wrong | When in doubt, escalate |

---

## Gary V decides without escalating (routine approvals)

| Situation | Gary V action |
|-----------|--------------|
| Routine evergreen teaching post | Approve if voice check passed |
| Standard gig announcement (date, venue, Humberto only) | Approve if factually correct |
| Personal reflection post with no third-party references | Approve if voice check passed |
| Repurposed version of a previously approved post | Approve if no new claims added |
| Queue management decisions (which post goes out when) | Gary V's call |
| Minor copy revision by Rosa | Gary V's call |

---

## Escalation format

When Gary V escalates to Humberto:

```
ESCALATION — [brief subject]

Post: [slug]
Status: [current stage]

What I need from you:
[One sentence — a yes/no decision, a factual check, or a content call]

The content:
[The post copy]

Options:
1. [Option A]
2. [Option B — or "Your preference"]
```

No long preamble. One clear ask.

## What is NOT an escalation

- A draft needing revision → that's a revision cycle (Maya → Rosa → Maya)
- A scheduling gap → that's Gary V's job to fill
- An experiment result → Eva summarizes for Gary V; Gary V decides
- A platform algorithm change → Omar and Eva flag; Gary V adapts

---

_When in doubt: ask Humberto. One short message now is better than a wrong post live._
